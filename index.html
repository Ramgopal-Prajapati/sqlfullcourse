CHAPTER 01
Introduction to Databases & SQL
Understand what data, information, databases and DBMS are — the foundation of everything

What is Data?

Definition

Data is raw, unprocessed facts and figures. Every single recorded value — a number, a name, a date — is data. By itself, data has no meaning without context.

Think of data like raw ingredients: flour, sugar, eggs. On their own, they're just raw materials with no particular meaning. Data works the same way.

🔢

Numeric Data

25, 60000, 9753528324 — raw numbers with no context

🔤

Text Data

"Ram", "Delhi", "Python" — raw string values

📅

Date/Time Data

"2024-06-21", "09:30:23" — raw timestamp values

What is Information?

Definition

Information is processed, organized data that has meaning and is useful for decision-making. When raw data is given context, it becomes information.

💡

Example: "25" is just data. But "Ram Sir is 25 years old, earns ₹60,000/month and lives in Bhopal" — that's information. The same values now have context and meaning.

What is a Database?

Definition

A Database is an organized, structured collection of related data stored in a system. It is the place where we store, manage, and retrieve data and perform CRUD operations — Create, Read, Update, Delete.

🏦 Real-World Example

A bank database stores: customer names, account numbers, balances, transaction history, branch locations. All organized so it can be retrieved and updated instantly — millions of records, milliseconds of response time.

⚡ CRUD Operations

  • Create — INSERT new records into tables
  • Read — SELECT and retrieve records
  • Update — UPDATE existing records
  • Delete — DELETE records from tables

What is DBMS?

Definition

A DBMS (Database Management System) is software that provides tools to create, manage, and access databases digitally. It acts as the bridge between the user/application and the actual stored data.

✅

Analogy: A library has thousands of books (Data). The building is the Database. The librarian + catalog system is the DBMS — it knows exactly where every book is, controls who can access what, and manages additions and removals efficiently.

Two Types of Databases

🗃️ Relational Database — SQL

Stores data in rows and columns like an Excel spreadsheet. Tables are linked through keys. Data is highly structured and follows a defined schema.

📦 Tool: MySQL Workbench

Examples: MySQL, PostgreSQL, Oracle, SQL Server

📄 Non-Relational Database — NoSQL

Stores data as documents, objects, or key-value pairs — like JSON. Flexible schema, great for unstructured or rapidly changing data.

📦 Tool: MongoDB

Examples: MongoDB, Firebase, Cassandra, Redis

What is SQL?

Definition

SQL (Structured Query Language) is a standard query language used to interact with relational databases. It allows creating tables, inserting data, retrieving information, updating records, and deleting data — all using simple English-like commands.

SQL Constraints

Definition

Constraints are rules assigned to table columns that restrict what data can be inserted. They maintain data integrity and prevent invalid or inconsistent records from entering the database.

PRIMARY KEY

Uniquely identifies every row. Cannot be NULL or duplicate. Each table should have exactly one primary key.

FOREIGN KEY

Creates a relationship between two tables. The value must exist as a primary key in the referenced table.

UNIQUE

Ensures all values in a column are different from each other. A table can have multiple unique columns.

NOT NULL

Prevents a column from having a NULL (empty) value. Data must always be provided for that column.

CHECK

Validates data against a condition before inserting. Example: age > 10, stock > 200, salary >= 5000.

DEFAULT

Assigns a default value automatically when no value is provided during an INSERT statement.

SQL — All Constraints in Action
-- PRIMARY KEY + NOT NULL + UNIQUE + CHECK + DEFAULT
CREATE TABLE employees (
    emp_id   INT         PRIMARY KEY,
    name     VARCHAR(50) NOT NULL,
    mobile   BIGINT      UNIQUE,
    age      INT         CHECK(age >= 18),
    status   VARCHAR(10) DEFAULT 'active',
    salary   INT         DEFAULT 5000
);

-- FOREIGN KEY: department table referenced by employee
CREATE TABLE departments (
    dept_id   INT         PRIMARY KEY,
    dept_name VARCHAR(50)
);

CREATE TABLE emp_dept (
    e_id   INT         PRIMARY KEY,
    d_id   INT,
    e_name VARCHAR(50),
    FOREIGN KEY (d_id) REFERENCES departments(dept_id)
);

Filter, compare and search data with precision using operators and conditions

Operators are special symbols that perform operations on column values or data. The WHERE clause uses operators to filter rows based on conditions — it acts like a gatekeeper that only lets matching records through.

⚖️ Comparison Operators

🧠 Logical / Special

AND  OR  NOT
BETWEEN  IN

SQL — All Operators & WHERE Examples
-- Setup
CREATE TABLE Students(ID INT, Name VARCHAR(50), Age INT, Marks INT, Grade CHAR(1), City VARCHAR(50));
INSERT INTO Students VALUES
(1,'Amit',18,90,'A','Delhi'),(2,'Neha',21,85,'B','Mumbai'),
(3,'John',17,60,'C','New York'),(4,'Sara',22,NULL,'F','London'),
(5,'Raj',19,100,'A',NULL);

-- Arithmetic: Calculate bonus marks
SELECT Name, Marks, Marks+5 AS Add_5, Marks*2 AS Double, Marks%10 AS Mod10
FROM Students;

-- Comparison Operators
SELECT * FROM Students WHERE Marks = 90;      -- Equal to
SELECT * FROM Students WHERE Marks > 80;      -- Greater than
SELECT * FROM Students WHERE Marks != 100;    -- Not equal
SELECT * FROM Students WHERE Marks <= 85;     -- Less than or equal

-- Logical Operators
SELECT * FROM Students WHERE Marks >= 85 AND Grade = 'A';
SELECT * FROM Students WHERE Marks < 85 OR Grade = 'C';
SELECT * FROM Students WHERE NOT Grade = 'F';

-- BETWEEN: Inclusive range
SELECT * FROM Students WHERE Marks BETWEEN 80 AND 100;

-- IN: Match from a list of values
SELECT * FROM Students WHERE City IN ('Delhi', 'Mumbai');

-- LIKE: Pattern matching with wildcards
SELECT * FROM Students WHERE Name LIKE 'A%';    -- Starts with A
SELECT * FROM Students WHERE Name LIKE '%a';    -- Ends with a
SELECT * FROM Students WHERE Name LIKE '%oh%';  -- Contains 'oh'

-- IS NULL / IS NOT NULL
SELECT * FROM Students WHERE Marks IS NULL;
SELECT * FROM Students WHERE City IS NOT NULL;

-- ORDER BY + LIMIT
SELECT * FROM Students ORDER BY Marks DESC LIMIT 3;

Summarize and analyze data using powerful mathematical functions

Aggregate Functions are built-in SQL functions that calculate a single result from multiple rows. They are the backbone of data analysis — used to count records, find averages, totals, minimums and maximums across entire datasets or groups.

➕

📏

AVG()

Average of values

⬇️

Minimum value

⬆️

Maximum value

SQL — Aggregate Functions + GROUP BY + HAVING
CREATE TABLE employees (
    emp_id INT PRIMARY KEY, emp_name VARCHAR(100),
    department VARCHAR(50), salary DECIMAL(10,2)
);
INSERT INTO employees VALUES
(1,'John','Sales',50000),(2,'Jane','IT',75000),
(3,'Mike','Sales',55000),(4,'Sarah','HR',60000),
(5,'David','IT',80000),(6,'Emily','HR',58000),
(7,'Robert','Sales',52000);

-- COUNT: How many employees total?
SELECT COUNT(*) AS total_employees FROM employees;

-- SUM: Total salary bill
SELECT SUM(salary) AS total_salary FROM employees;

-- AVG: Average salary
SELECT AVG(salary) AS avg_salary FROM employees;

-- MIN and MAX
SELECT MIN(salary) AS lowest, MAX(salary) AS highest FROM employees;

-- GROUP BY: Per department analysis
SELECT department, COUNT(*) AS emp_count,
       AVG(salary) AS avg_salary, SUM(salary) AS total_salary
FROM employees
GROUP BY department;

-- HAVING: Filter groups where avg salary > 60000
SELECT department, AVG(salary) AS avg_sal
FROM employees
GROUP BY department
HAVING AVG(salary) > 60000;

CHAPTER 05
DML, DDL & DQL Commands
The three pillars of SQL — manipulate data, define structure, and query results

🔍 DQL — Data Query Language

Used to retrieve and filter data from tables. Includes SELECT with all its powerful clauses for filtering, sorting, grouping.

SELECT

✏️ DML — Data Manipulation Language

Used to add, change, or remove rows inside tables. Changes can be rolled back with TCL commands.

INSERT | UPDATE | DELETE

🏗️ DDL — Data Definition Language

Used to define and change the structure of database objects. DDL changes are auto-committed.

CREATE | ALTER | DROP | TRUNCATE

SQL — DML: INSERT, UPDATE, DELETE
-- INSERT: Add a single row
INSERT INTO Students VALUES(1, 'Amit Sharma', 22, 'Python', 12000, 'Delhi');

-- INSERT: Multiple rows at once (efficient)
INSERT INTO Students VALUES
(2, 'Sneha Patel', 20, 'Java',    10000, 'Mumbai'),
(3, 'Rahul Mehta', 21, 'Python',  0,     'Delhi'),
(4, 'Neha Gupta',  19, 'Web Dev', 15000, 'Bhopal');

-- UPDATE: Change one field
UPDATE Students SET FeesPaid = 13000 WHERE StudentID = 1;

-- UPDATE: Change multiple fields
UPDATE Students SET Age = 24, City = 'Noida' WHERE FullName = 'Sneha Patel';

-- UPDATE: Conditional — reset fees for Python students who haven't paid
UPDATE Students SET FeesPaid = 15000
WHERE Course = 'Python' AND FeesPaid = 0;

-- UPDATE: Increase Java fees by 10%
UPDATE Students SET FeesPaid = FeesPaid * 1.1 WHERE Course = 'Java';

-- DELETE: Remove a specific row
DELETE FROM Students WHERE StudentID = 5;

-- DELETE: Remove all rows with zero fees
DELETE FROM Students WHERE FeesPaid = 0;

-- TRUNCATE: Remove ALL rows instantly (cannot rollback)
TRUNCATE TABLE Students;
SQL — DDL: CREATE, ALTER, DROP
-- CREATE: Build a new table
CREATE TABLE Students (
    StudentID INT PRIMARY KEY, FullName VARCHAR(100),
    Age INT, Course VARCHAR(50), JoinDate DATE, FeesPaid DECIMAL(10,2)
);

-- ALTER: Add a new column
ALTER TABLE Students ADD Email VARCHAR(100);

-- ALTER: Modify column data type
ALTER TABLE Students MODIFY Age SMALLINT;

-- ALTER: Remove a column
ALTER TABLE Students DROP COLUMN Email;

-- ALTER: Rename a column (MySQL syntax)
ALTER TABLE Students CHANGE FullName StudentName VARCHAR(50);

-- ALTER: Add a CHECK constraint
ALTER TABLE Students ADD CONSTRAINT chk_age CHECK(Age >= 18);

-- RENAME: Rename entire table
RENAME TABLE Students TO StudentRecords;

-- DROP: Delete entire table (structure + data)
DROP TABLE StudentRecords;
✅

DELETE vs TRUNCATE vs DROP: DELETE removes specific rows and can be rolled back. TRUNCATE clears ALL rows instantly — faster but cannot be rolled back. DROP removes the entire table including its structure — gone forever.

SQL Joins — Combining Tables
Connect related tables and retrieve combined data — visualized with Venn Diagrams

Definition

A JOIN combines rows from two or more tables based on a related column — usually a Foreign Key. Instead of storing everything in one giant table, we split data into organized tables and use JOINs to bring it together when needed.

All Join Types — Venn Diagram Visualization

🔵 INNER JOIN

Returns rows where both tables have matching records. Non-matching rows from either side are excluded.

INNER JOIN / JOIN

⬅️ LEFT JOIN

All rows from LEFT table + matching rows from right. Non-matching right rows appear as NULL.

LEFT JOIN / LEFT OUTER JOIN

➡️ RIGHT JOIN

Matching rows from left + ALL rows from RIGHT table. Non-matching left rows appear as NULL.

RIGHT JOIN / RIGHT OUTER JOIN

🔄 FULL JOIN

ALL rows from BOTH tables. NULLs where there's no match. MySQL: use LEFT + RIGHT JOIN with UNION.

FULL OUTER JOIN (via UNION)

✖️ CROSS JOIN

Every row from table A combined with every row from table B. 5 employees × 4 departments = 20 rows.

CROSS JOIN

🔁 SELF JOIN

A table joined to itself. Perfect for hierarchical relationships like employee → manager.

E JOIN E AS M ON E.ManagerID = M.EmpID
SQL — All JOIN Types with Data
-- Setup: Departments Table
CREATE TABLE Departments(DeptID INT PRIMARY KEY, DeptName VARCHAR(100));
INSERT INTO Departments VALUES(1,'HR'),(2,'IT'),(3,'Finance'),(4,'Marketing');

-- Setup: Employees Table
CREATE TABLE Employees(EmpID INT PRIMARY KEY, EmpName VARCHAR(100), DeptID INT);
INSERT INTO Employees VALUES
(101,'Amit Sharma',1),(102,'Priya Singh',2),
(103,'Ravi Kumar',2),(104,'Neha Gupta',NULL),  -- No department
(105,'Anil Yadav',5);                             -- Invalid dept

-- 1. INNER JOIN: Only employees with valid departments
SELECT E.EmpName, D.DeptName
FROM Employees E INNER JOIN Departments D ON E.DeptID = D.DeptID;

-- 2. LEFT JOIN: All employees, NULLs for unmatched department
SELECT E.EmpName, D.DeptName
FROM Employees E LEFT JOIN Departments D ON E.DeptID = D.DeptID;

-- 3. RIGHT JOIN: All departments, even empty ones
SELECT E.EmpName, D.DeptName
FROM Employees E RIGHT JOIN Departments D ON E.DeptID = D.DeptID;

-- 4. FULL JOIN (MySQL — using UNION)
SELECT E.EmpName, D.DeptName
FROM Employees E LEFT JOIN Departments D ON E.DeptID = D.DeptID
UNION
SELECT E.EmpName, D.DeptName
FROM Employees E RIGHT JOIN Departments D ON E.DeptID = D.DeptID;

-- 5. CROSS JOIN: Every employee x every department
SELECT E.EmpName, D.DeptName FROM Employees E CROSS JOIN Departments D;

-- 6. SELF JOIN: Employee and their manager
SELECT E.EmpName AS Employee, M.EmpName AS Manager
FROM Employees E LEFT JOIN Employees M ON E.ManagerID = M.EmpID;

-- 7. 3-Table JOIN: Students + Courses + Enrollments
SELECT E.EnrollmentID, S.StudentName, C.CourseName, E.EnrollmentDate
FROM Enrollments E
JOIN Students S ON E.StudentID = S.StudentID
JOIN Courses C  ON E.CourseID  = C.CourseID;

Subqueries — Nested Queries
A query inside another query — understand how parent and child queries work together

Definition

A Subquery (also called a Nested Query) is a SQL query written inside another SQL query. The inner (child) query executes first, produces a result, and that result is then used by the outer (parent) query. This allows solving complex problems step by step.

Parent vs Child Query — Visual Breakdown

PARENT QUERY — Outer (runs second)
SELECT emp_name, salary FROM employees
WHERE salary > ( ← uses child's result here )
↓ Parent waits for child to finish executing
CHILD QUERY — Inner (runs first)
SELECT AVG(salary) FROM employees
▶ Child Output: 65000 (average salary of all employees)

▶ Parent Final Output — Employees where salary > 65000:

emp_namesalary
Priya70000
Asha90000
Karan70000
SQL — All Types of Subqueries
-- SETUP
INSERT INTO employees VALUES
(1,'Ram',50000,101),(2,'Priya',70000,102),
(3,'John',60000,101),(4,'Asha',90000,103),
(5,'Karan',70000,102),(6,'Meena',50000,103);

-- 1. SINGLE-ROW subquery (returns one value)
--    Employees earning more than average salary
SELECT emp_name, salary FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- 2. MULTIPLE-ROW subquery (returns a list)
--    Employees in IT or HR departments
SELECT emp_name FROM employees
WHERE dept_id IN (
    SELECT dept_id FROM departments
    WHERE dept_name IN ('IT', 'HR')
);

-- 3. CORRELATED subquery (inner uses outer's value)
--    Highest paid employee in EACH department
SELECT emp_name, salary, dept_id
FROM employees e
WHERE salary = (
    SELECT MAX(salary)
    FROM employees
    WHERE dept_id = e.dept_id   -- ← references outer query's row
);

-- 4. SCALAR subquery (used in SELECT column)
--    Each employee's salary vs company average
SELECT emp_name, salary,
    (SELECT AVG(salary) FROM employees) AS company_avg,
    salary - (SELECT AVG(salary) FROM employees) AS difference
FROM employees;

-- 5. Subquery in FROM (Inline View / Derived Table)
SELECT dept_id, avg_salary
FROM (
    SELECT dept_id, AVG(salary) AS avg_salary
    FROM employees
    GROUP BY dept_id
) AS dept_summary;

-- 6. SECOND HIGHEST SALARY (classic interview question!)
SELECT MAX(salary) AS second_highest
FROM employees
WHERE salary < (SELECT MAX(salary) FROM employees);

TCL — Transaction Control Language

Definition

TCL (Transaction Control Language) manages database transactions. A transaction is a group of SQL statements executed as one atomic unit — either all statements succeed together, or all fail together. This ensures data stays consistent even if something goes wrong mid-way.

SQL — TCL: COMMIT, ROLLBACK, SAVEPOINT
-- Example 1: COMMIT — Bank transfer Alice → Bob ₹1000
BEGIN;

UPDATE BankAccounts SET Balance = Balance - 1000 WHERE AccountID = 1; -- Debit Alice
UPDATE BankAccounts SET Balance = Balance + 1000 WHERE AccountID = 2; -- Credit Bob

COMMIT; -- ✅ Save both changes permanently

-- Example 2: ROLLBACK — Cancel if error occurs
BEGIN;

UPDATE BankAccounts SET Balance = Balance - 2000 WHERE AccountID = 1;
-- Error detected! Alice doesn't have ₹2000
ROLLBACK; -- ↩️ Undo everything — Alice's balance is unchanged

-- Example 3: SAVEPOINT — Partial undo
BEGIN;

UPDATE BankAccounts SET Balance = Balance - 500 WHERE AccountID = 1;
SAVEPOINT checkpoint_a; -- 📌 Save this point

UPDATE BankAccounts SET Balance = Balance + 500 WHERE AccountID = 2;

ROLLBACK TO checkpoint_a; -- ↩️ Undo Bob's credit, keep Alice's debit

COMMIT; -- Only Alice's ₹500 deduction saved

DCL — Data Control Language

Definition

DCL (Data Control Language) controls who can access what in the database. It has two commands: GRANT (give permissions to users) and REVOKE (take away permissions). These require admin/DBA privileges.

SQL — DCL: GRANT & REVOKE
-- Step 1: Create a new user
CREATE USER 'trainer'@'localhost' IDENTIFIED BY 'pass123';

-- Step 2: GRANT select-only access
GRANT SELECT ON EmployeeData TO 'trainer'@'localhost';

-- Step 3: GRANT multiple permissions
GRANT SELECT, INSERT, UPDATE ON EmployeeData TO 'trainer'@'localhost';

-- Step 4: GRANT full admin access (use carefully!)
GRANT ALL PRIVILEGES ON *.* TO 'trainer'@'localhost';

-- Step 5: REVOKE specific permissions
REVOKE INSERT, UPDATE ON EmployeeData FROM 'trainer'@'localhost';

-- Step 6: REVOKE all permissions
REVOKE ALL PRIVILEGES ON *.* FROM 'trainer'@'localhost';

-- Step 7: Check what permissions a user has
SHOW GRANTS FOR 'trainer'@'localhost';

-- Step 8: Apply changes immediately
FLUSH PRIVILEGES;

-- Step 9: Remove the user completely
DROP USER 'trainer'@'localhost';

✅ COMMIT — Permanently Saved

Once COMMIT is executed, all changes in the transaction are written permanently to the database. They cannot be undone with ROLLBACK after this point.

↩️ ROLLBACK — Undo All Changes

ROLLBACK cancels all changes made since the last COMMIT or SAVEPOINT. The database is restored to its previous state — as if the transaction never happened.
